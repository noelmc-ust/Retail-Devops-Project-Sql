1. Project Overview & Guiding Principles

NeoBank is a simplified digital banking platform implemented as 12 Node.js microservices behind a React + Vite single-page application. Every service owns its own database, communicates synchronously over REST (service-to-service via internal hostnames), and asynchronously via RabbitMQ for events like "payment completed" or "KYC approved". The entire stack runs in Docker containers and is orchestrated by Kubernetes for your DevOps pipeline practice.

Universal Standards — Every Service Must Follow

Single Responsibility

Each service owns exactly one business domain. No service reaches into another service's database directly — only via HTTP calls or events.

Database Per Service

Each service has its own DB instance (PostgreSQL or MongoDB). No shared schemas, no cross-DB joins. Data is replicated via events when needed.

12-Factor App

Config via ENV variables, stateless processes, logs to stdout as JSON, health check endpoints at /health, graceful shutdown on SIGTERM.

Consistent Error Shape

All APIs return { success, data, error: { code, message } }. HTTP status codes follow REST conventions strictly.

Observability

Every service exposes /metrics (Prometheus format), emits structured JSON logs with requestId, service, level, timestamp.

Security by Default

All routes require a valid JWT (verified at gateway). Internal service calls use a shared INTERNAL_SECRET header. Passwords are bcrypt-hashed, sensitive data is masked in logs.

Idempotency

Payment and transaction endpoints accept an Idempotency-Key header. Re-sending the same request returns the original response without creating a duplicate record.

Code Style

ESLint + Prettier enforced via pre-commit hook (Husky). Airbnb ruleset. All files use ES modules (import/export). JSDoc on all exported functions.

Axios Client Setup (api/client.js)

import axios from 'axios';
import { useAuthStore } from '../store/auth.store';

const client = axios.create({
  baseURL: import.meta.env.VITE_API_URL || 'http://localhost:3000/api',
  timeout: 10000,
  headers: { 'Content-Type': 'application/json' },
});

// Attach JWT to every request
client.interceptors.request.use((config) => {
  const token = useAuthStore.getState().accessToken;
  if (token) config.headers.Authorization = `Bearer ${token}`;
  return config;
});

// Auto-refresh on 401
client.interceptors.response.use(
  (res) => res.data,
  async (err) => {
    if (err.response?.status === 401) {
      await useAuthStore.getState().refreshTokens();
      return client(err.config);
    }
    return Promise.reject(err.response?.data?.error || err);
  }
);

export default client;

1. API Gateway

Single entry point. Routes all client traffic, enforces auth, rate-limits, and transforms errors into a unified shape.

Port :3000 Member 1 MongoDB (request logs) Redis (rate limit counters)

Responsibilities

Receive all HTTP requests from the frontend. Validate JWT (RS256) on protected routes. Rate-limit by IP (100 req/min). Reverse-proxy to downstream services using http-proxy-middleware. Append X-Request-Id, X-User-Id, X-User-Role headers before forwarding. Log every request/response to MongoDB. Return standardised error JSON if a downstream service is unreachable.

NPM Dependencies

express, http-proxy-middleware, jsonwebtoken, express-rate-limit,
redis, ioredis, mongoose, morgan, winston, helmet, cors, uuid,

Manages bank accounts — savings, current, fixed deposit. Controls balance state and exposes balance enquiry APIs.

Port :3003 Member 2 PostgreSQL

Responsibilities

Create and manage bank accounts. Maintain balances with optimistic locking to prevent race conditions. The Transaction Service calls this service to debit/credit balances — the Account Service is the single source of truth for all balances. Publish events when account state changes.

POST/accounts/:id/creditInternal — called by Transaction Service

PostgreSQL Schema

CREATE TABLE accounts (
  id           UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id      UUID NOT NULL,
  account_no   VARCHAR(16) NOT NULL UNIQUE,  -- generated: 16-digit
  ifsc_code    VARCHAR(11) DEFAULT 'NBNK0000001',
  type         VARCHAR(20) NOT NULL,          -- savings|current|fd
  balance      NUMERIC(15,2) DEFAULT 0.00,
  currency     CHAR(3) DEFAULT 'USD',
  status       VARCHAR(20) DEFAULT 'active', -- active|frozen|closed
  interest_rate NUMERIC(5,2),
  version      INTEGER DEFAULT 0,            -- optimistic lock
  created_at   TIMESTAMPTZ DEFAULT NOW(),
  updated_at   TIMESTAMPTZ DEFAULT NOW()
);

Optimistic Lock Pattern (debit endpoint)

-- Atomic debit with optimistic locking
UPDATE accounts
SET balance = balance - $1,
    version = version + 1,
    updated_at = NOW()
WHERE id = $2
  AND version = $3        -- must match current version
  AND balance >= $1       -- sufficient funds check
  AND status = 'active'
RETURNING *;

5. Transaction Service

Ledger of all money movements. Records every debit/credit with full audit trail. Orchestrates balance updates via Account Service.

Port :3004 Member 3 PostgreSQL (append-only ledger)

Responsibilities

Create immutable transaction records. Coordinate two-phase operations (debit source → credit destination) using a saga-like pattern — if credit fails, publish a compensating debit-reversal event. Provide paginated transaction history. Support filtering by date range, type, and amount. Generate transaction references (UTR numbers).

API Endpoints

MethodPathNotes
GET/transactions?accountId=&page=&limit=&from=&to=&type=
GET/transactions/:idSingle transaction detail
POST/transactions/internal-transferBetween own accounts
GET/transactions/statement?accountId=&month= → PDF/JSON statement

PostgreSQL Schema (append-only)

CREATE TABLE transactions (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  reference_no    VARCHAR(25) NOT NULL UNIQUE,  -- UTR/TXN number
  from_account_id UUID,
  to_account_id   UUID,
  amount          NUMERIC(15,2) NOT NULL,
  currency        CHAR(3) DEFAULT 'USD',
  type            VARCHAR(30) NOT NULL,  -- transfer|payment|loan_emi|interest|fee
  status          VARCHAR(20) DEFAULT 'completed', -- pending|completed|failed|reversed
  description     VARCHAR(255),
  metadata        JSONB,                 -- payment gateway ref, etc.
  idempotency_key VARCHAR(100) UNIQUE,   -- prevent duplicate posting
  created_at      TIMESTAMPTZ DEFAULT NOW()
);
-- NO UPDATE or DELETE — append-only ledger
-- Reversals are NEW rows with type='reversal'

7. Loan Service

Loan application, approval workflow, EMI schedule generation, and repayment tracking.

Port :3006 Member 4 PostgreSQL

Responsibilities

Handle loan application lifecycle: apply → under_review → approved/rejected → active → closed. Calculate EMI using the standard formula: EMI = P * r * (1+r)^n / ((1+r)^n - 1) where r = monthly rate. Generate full repayment schedule. Track missed EMIs. Disburse to account via Transaction Service on approval. Debit EMIs on due date via a scheduled cron job.

Caching Pattern

async function getRates(base = 'USD') {
  const cacheKey = `fx:rates:${base}`;
  const cached = await redis.get(cacheKey);
  if (cached) return JSON.parse(cached);

  const rates = await fetchFromExternalAPI(base);
  await redis.setex(cacheKey, 60, JSON.stringify(rates)); // 60s TTL
  await persistDailyRate(base, rates);                      // PostgreSQL
  return rates;
}

GitHub Actions CI/CD Pipeline

# .github/workflows/ci.yml
name: CI/CD Pipeline
on:
  push: { branches: [main, develop] }
  pull_request: { branches: [main] }

jobs:
  test:
    runs-on: ubuntu-latest
    strategy:
      matrix: { service: [api-gateway, auth-service, user-service, ...] }
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: { node-version: '20', cache: 'npm' }
      - run: cd services/${{ matrix.service }} && npm ci && npm test

  build-push:
    needs: test
    if: github.ref == 'refs/heads/main'
    runs-on: ubuntu-latest
    steps:
      - uses: docker/login-action@v3
        with: { username: ${{ secrets.DOCKER_USER }}, password: ${{ secrets.DOCKER_TOKEN }} }
      - run: |
          docker build -t neobank/${{ matrix.service }}:${{ github.sha }} \
            ./services/${{ matrix.service }}
          docker push neobank/${{ matrix.service }}:${{ github.sha }}

  deploy:
    needs: build-push
    runs-on: ubuntu-latest
    steps:
      - uses: azure/k8s-deploy@v4   # or kubectl apply
        with:
          manifests: infra/k8s/*.yaml
          images: neobank/${{ matrix.service }}:${{ github.sha }}

Monitoring Stack

Prometheus scrapes /metrics from all services every 15s. Grafana dashboards: request rate, error rate, latency (p50/p95/p99), DB connection pool, RabbitMQ queue depth. Alert rules: error rate > 5% for 2 min, p95 latency > 2s, any service health check failing.

Shared RabbitMQ Setup

Exchange: neobank.events (topic type). Dead-letter exchange: neobank.dlx. Each consumer service declares its own queue with x-dead-letter-exchange. Retry policy: 3 attempts with exponential backoff before routing to DLQ.

Local Development

make up → starts all infra (DB, Redis, RabbitMQ, Mailhog). make dev-{service} → runs single service with nodemon. VS Code workspace file provided. Shared .env.example at root. Seed scripts under infra/seeds/.

Security Checklist

Helmet.js on all services. CORS restricted to frontend origin. Secrets in k8s Secrets / .env (never in code). Bcrypt rounds = 12. JWT RS256 with 15-min access token. Internal service calls validated by X-Internal-Secret header. SQL parameterised queries only (no string concat).
